# Agent Scenario Matrix

This reference supports the delegated agent workflow lifecycle.

Use this reference when running live workflow scenarios.

1. Define at least five realistic scenarios.
2. Include happy path, variant, verification, recovery, and setup buckets.
3. Record exact inputs used for each scenario.
4. Record PASS, PARTIAL, FAIL, or BLOCKED for every scenario.
5. Capture the friction point in one sentence.
6. Capture the follow-up action needed to remove that friction.
7. Include one scenario that stresses portability or alternate-runtime use.
8. Include one scenario that proves the safety guardrail is understandable.
9. Keep the evidence concise but specific.
10. Do not collapse PARTIAL or BLOCKED outcomes into PASS.
