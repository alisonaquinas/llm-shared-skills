# Command Command Scope Template

This reference supports the reusable command workflow lifecycle.

Use this template to plan a reusable command workflow before design begins.

1. Name the workflow and its primary trigger phrases.
2. Record the exact problem the workflow solves.
3. Define required inputs, optional inputs, and expected outputs.
4. List dependencies such as scripts, config files, or local tools.
5. State explicit non-goals so scope does not drift during authoring.
6. Capture one happy-path scenario and one recovery scenario.
7. Document who will invoke the workflow and under which constraints.
8. Record rollback or manual fallback steps.
9. Mark assumptions that must be rechecked in design.
10. Do not leave any trigger or success criterion as TBD.
