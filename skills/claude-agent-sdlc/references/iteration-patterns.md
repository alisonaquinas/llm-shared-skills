# Agent Iteration Patterns

This reference supports the delegated agent workflows SDLC.

Iteration defines the shortest path back to green after a failure.

1. Re-enter planning for scope or audience mistakes.
2. Re-enter design for role or safety mistakes.
3. Re-enter creation for runtime defects or missing artifacts.
4. Re-enter testing for coverage gaps.
5. Re-enter verification when the implementation and contract drift apart.
6. Re-enter integration when discovery or placement breaks.
7. Re-enter validation when only the score or evidence is incomplete.
8. Re-enter test drive when live scenarios reveal unresolved friction.
9. Record which gate failed and why before rework starts.
10. Never skip the failed gate on the next pass.
