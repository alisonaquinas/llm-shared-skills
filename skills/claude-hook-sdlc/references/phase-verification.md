# Hook Phase Verification

This reference supports the hook automation rules SDLC.

Verification confirms the implemented rule still matches the approved contract.

1. Recheck events, matchers, defaults, and outputs.
2. Recheck required files and helpers.
3. Recheck safety rules and fallback paths.
4. Recheck portability notes and examples.
5. Treat mismatches as upstream defects rather than documentation edits.
6. Use the verification skill to run a repeatable checklist.
7. Re-run verification after meaningful creation changes.
8. Keep the checklist results explicit and reviewable.
9. Do not integrate a rule with an unresolved verification failure.
10. Verification is the final static gate before live wiring.
