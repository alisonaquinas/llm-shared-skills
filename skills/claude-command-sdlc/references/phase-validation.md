# Command Phase Validation

This reference supports the command workflows SDLC.

Validation scores whether the workflow is ready for reuse beyond a single local fix.

1. Score discoverability, safety, portability, and operability.
2. Base the verdict on the planning, verification, and test-drive evidence.
3. Require safety to PASS before approving.
4. Record WARN items clearly even when the verdict is APPROVE.
5. Use the validation skill for rubric-driven review.
6. Treat repeated WARN patterns as a signal to improve the family template.
7. Keep the scoring language concrete and evidence-based.
8. Validation should not invent new interface requirements.
9. Loop back immediately on FAIL.
10. Ship only after validation evidence is written.
