# Command Phase Creation

This reference supports the command workflows SDLC.

Creation turns the approved design into real workflow artifacts.

1. Build the required files in a predictable structure.
2. Add helper scripts or templates only when they reduce repeated effort.
3. Keep filenames and labels aligned with the design contract.
4. Run a smoke check as soon as the files exist.
5. Document runtime-specific config beside the implementation.
6. Stop and revisit design if implementation needs a new public decision.
7. Use the creation skill for scaffolding and implementation loops.
8. Keep artifacts deterministic and easy to rerun.
9. Capture a minimal handoff note for downstream testing.
10. Creation is complete only when the workflow can execute locally.
