# Verification Checklist

Use this checklist after editing files.

- Relevant repo guidance was checked before editing.
- The change stayed within the intended scope.
- Structured formats used the safest available approach.
- `jq` was preferred for JSON when applicable.
- `yq` was preferred for YAML, XML, or TOML when applicable.
- Formatting or normalization was run when relevant.
- Lint, type, or static checks were run when relevant.
- Focused tests were run for affected behavior when available.
- The diff was reviewed for unrelated churn.
- Remaining risks, assumptions, or skipped checks were reported.
